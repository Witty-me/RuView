# 竞品仓库外宣真实性评估报告（技术主线提纲）

> 适用对象：管理层 + 技术负责人
>
> 使用场景：判断竞品宣传是否可信、技术上是否可落地、我方是否需要投入对标

## 0. 一页技术结论（先看）
- **结论一句话**：该竞品“外宣强、落地弱”的风险较高，需以可复现实机证据为唯一判据。
- **核心判断**：
  - 宣传中的关键能力（如穿墙感知、DensePose 级输出、可商用部署）必须逐条技术核验。
  - 若存在“实机不可跑”或“核心链路依赖 mock”，对应宣传项直接降级为高风险存疑。
- **决策原则**：我方不按热度投入，只按“可运行证据 + 复现结果”投入。

---

## 1. 报告目标（技术优先）
### 1.1 本报告不做什么
- 不参与竞品项目开发，不给对方优化建议。
- 不写过多宣传流程建议，避免偏离技术判断主线。

### 1.2 本报告要回答什么
- **真实性**：宣传能力在代码与实机层面是否成立。
- **可落地性**：是否具备连续运行与稳定输出能力。
- **借鉴意义**：哪些技术路线值得我方投入、哪些应暂缓。

## 3. 真实性判定框架（技术判定）
### 3.1 四级标签
- **A 已证实**：可在目标场景跑通，且可复现。
- **B 部分属实**：有实现，但关键链路缺失。
- **C 高风险存疑**：以 mock/占位为主，无法证明可用。
- **D 不成立/明显夸大**：与仓库证据冲突。

### 3.2 两个“一票否决项”
- **否决项 1：实机不可跑**（真实硬件无法稳定产出）
- **否决项 2：核心能力靠 mock**（高级功能无真实实现）

> 任一命中，宣传项默认降级为 C 或 D。

---

## 4. 外宣真实性“打假清单”（报告主体）
> 建议占比 50% 以上。

### 4.1 宣传点清单（示例）
- P1：无需摄像头，准确监控人体位置/姿态
- P2：可穿墙监测人体状态
- P3：项目已具备工程化部署能力
- P4：高级 AI 功能已可用（DensePose 级输出）
- P5：高 star/高热度说明技术已成熟

### 4.2 每条宣传点的固定分析模板
1. 外宣原话（来源+摘录）
2. 成立条件（数据、模型、实机）
3. 仓库证据（支持/反证）
4. 一票否决项是否命中
5. 判定等级（A/B/C/D）
6. 一句话结论（技术与交付风险）
7. 沟通备注（可选，1-2句）：仅说明表述边界，不扩展流程内容

### 4.3 必须明确的反证类型
- 哪些能力**不能在实机跑起来**。
- 哪些能力**只在接口/文档层存在**。
- 哪些结果**疑似由 mock 驱动**。

## 5. 借鉴价值与技术路线判别（结合我方现状）
### 5.1 可借鉴的短中期事项（优先）
1. **工程化与可观测性**（服务分层、日志、监控、健康检查）
   - 借鉴意义：提升部署稳定性与交付可信度。
   - 难度：低-中。
2. **数据闭环基础设施**（采集规范、标注流程、评测基线）
   - 借鉴意义：为复杂能力铺路。
   - 难度：中。
3. **能力分级表达机制**（实现中/验证中/愿景）
   - 借鉴意义：减少“叙事超前于实现”的风险。
   - 难度：低。

### 5.2 高价值但高难度事项（审慎投入）
1. **跨环境泛化（ADR-027 类）**
   - 难度：很高（多场景大数据 + 长周期迭代）。
   - 判别：无可复现实机证据前，不认定已做成。
2. **复杂姿态重建 / DensePose 级输出**
   - 难度：很高（数据质量、模型泛化、实时推理三重门槛）。
   - 判别：缺真实链路/权重/实测则视为概念验证。
3. **穿墙与复杂场景鲁棒性**
   - 难度：很高（墙体、多径、干扰、部署差异影响大）。
   - 判别：单点演示不等于可交付能力。

### 5.3 统一解释模板：为何我们没做出来、他们也大概率没做成
- **数据门槛**：复杂功能首先受限于高质量、多环境、可对齐数据。
- **泛化门槛**：实验室有效不代表真实场景稳定有效。
- **系统门槛**：采集、推理、时延、稳定性需同时满足。
- **验证门槛**：缺少公开可复现实机报告时，不能认定成熟。

### 5.4 我方建议路线（技术落地导向）
- 阶段 1：强化已落地能力（存在检测/测距）+ 稳定性指标化。
- 阶段 2：拓展中等难度可交付能力（受限场景人体状态识别）。
- 阶段 3：数据闭环成熟后，再评估跨环境泛化与复杂姿态重建。

## 6. 交付格式（技术主线版）
1. 第一页：技术结论板（真实性等级、可运行性、关键风险）
2. 第二页：宣传点真假总表（P1-P5，A/B/C/D + 否决项）
3. 第三页：关键反证页（实机不可跑 / mock 证据）
4. 第四页：借鉴价值与投入优先级（低/中/高难度）
5. 附录：证据索引（可追溯到代码/文档）

- D1：宣传语句抽取 + 初判
- D2：完成“实机不可跑 / mock”主证据页
- D3：完成借鉴价值与难度矩阵
- D4：完成四页技术汇报材料
